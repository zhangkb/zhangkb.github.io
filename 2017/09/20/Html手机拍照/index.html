关于网页手机拍照中出现的一些问题

17/2/22： 从调研开始到项目即将上线不得不说各个浏览器的对H5新api的兼容性不一而足。很难去做到尽善尽美。开始测试的时候出现了一堆 奇奇怪怪的问题，到最后也没能完整的解决列举如下：

  1. 在APP中的内置浏览器中无法长按保存图片。
  2. 在某些APP中无法调取相机进行拍照。
  3. 关于audio的自动播放问题，抱歉没能解决 依然有一部分机型出现问题。
  4. 对于某些机型会出现调取摄像头拍照后无法使用图片，这里没办法解决，使用相册存在的图片可以完成流程。

前言：项目是为了创建一个上传照片，合成，截图分享功能的移动端页面。
在网上看了很多一般都是通过canvas库完成的，由于本人技术有限所以使用了一种讨巧的 方法，前期的手势运动都是对图片的处理使用了hammerjs，完成以后使用html2canvas进行截取元素。由于demo写的太丑就不敢放了，出现了很多问题归总如下。

  1. 图片模糊问题：
    解决方案：

  1. canvas 生成图片

  2. input 上传图片预览

  3. 关于上传的照片经过html2canvas截取后IOS会翻转90°

对图片翻转信息的另一种处理方式：

  1. 安卓下 video 标签默认顶层
    解决办法：使用js 动态删除和添加controls属性，这个属性影响导致！有controls属性，

  2. 手机QQ浏览器下video控件会强制z-index最大无法解决

  3. canvas 出现的跨域问题。这是因为当你编辑图片时会因为某些图片或者其他元素的域名与当前域名不符合 造成的无法编辑事件。前端无法解决需要后台的配合。需要后台传递一个BASE64的图片信息。

  4. 关于IOS中微信无法自动播放音频的问题，这里可以引用微信的公共SDK：http://res.wx.qq.com/open/js/jweixin-1.0.0.js，但是某些机型可能依然无法播放所以需要添加触摸播放事件。

  5. 大致的问题就是这些，做得不够完善，希望会有更好的方法解决。
